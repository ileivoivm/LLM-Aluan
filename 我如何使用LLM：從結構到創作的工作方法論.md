# 我如何使用 LLM：從結構到創作的工作方法論

> **講者**：王新仁（阿亂）— 生成式藝術家、創意程式設計師、akaSwap 共同創辦人
> **工具**：Claude Code、Cursor、Cowork 模式
> **核心觀點**：作品只是表象，背後都是結構。留下錨點，讓一切可以被 index。

---

## 一、從 2021 年開始的思考：作品背後都是結構

2021 年，我在 Art Blocks 發表了第一件長篇生成藝術作品 **Good Vibrations**。表面上它是一件視覺動態藝術——旋轉的圓弧與線段在畫布上互動，交叉時觸發聲音。但我在裡面埋了一個隱藏機制：同時按下 A+S+D+F，畫面會切換成「樂譜模式」，揭示每個 token 背後的音樂結構。

```
tokenData.hash → seed → 確定性隨機數 → 所有參數（顏色、旋律數、波形、頻率）
```

每個 token 都是一首獨特的曲子，但只有知道快捷鍵的人才看得到樂譜。這讓我開始思考：**如果作品的本質是結構，那結構本身就可以被記錄、被還原、被重新生成。**

這個思考，後來成為我跟 LLM 協作的核心方法論。

## 二、核心方法論：JSON 作為記憶層

我跟 LLM 協作的基本邏輯是：**用結構化資料（JSON + Markdown）建立一個人與機器之間的協作協議。**

LLM 的上下文有限、記憶會斷。我的解法不是依賴模型記住一切，而是主動建立外部記憶——用 JSON 和 .md 檔案把所有重要資訊結構化，讓 LLM 每次都能從這些錨點重新理解脈絡。

以下是我在不同專案中的實踐：

### 2.1 小說創作：JSON 作為故事資料庫

**專案：《修仙-七玄關》**

寫長篇小說最大的問題是連貫性。角色在第三章說的話，不能跟第一章矛盾；伏筆埋了就要收。LLM 沒辦法記住幾萬字的前文，所以我建了一套 JSON 結構：

```json
{
  "chapter_1": {
    "segments": {
      "segment_1": {
        "title": "序曲－雨中的電話",
        "summary": "林承祐在深夜接到命案報告...",
        "content": "（完整文本）",
        "analysis": {
          "setting": "嘉義市區，林承祐的老公寓",
          "time": "午夜剛過",
          "characters": ["林承祐", "小陳"],
          "emotional_tone": "孤獨、麻木、恐懼",
          "foreshadowing": "宗教案件的陰影重現"
        }
      }
    }
  }
}
```

每個段落除了完整文本，還附帶 analysis：場景、時間、天氣、角色、情緒基調、文學手法、伏筆。這樣當我請 LLM 寫第五章時，它可以回溯前面的 analysis，確保情緒曲線連貫、伏筆被正確延續。

同時我用 .md 規劃角色設定、關係網絡、章節大綱，甚至建了一整套互動式網頁（HTML + Tailwind CSS + JSON），把人物關係圖、反派剖析、章節結構全部視覺化。

**關鍵心得**：不要期待 LLM 記住一切，主動幫它建索引。

### 2.2 生成藝術：JSON 作為確定性重播系統

**專案：InkField（墨域）**

InkField 是一套基於 p5.js + WebGL 的數位水墨繪畫引擎。它的錄製/播放系統達到 99.9% 一致性，核心就是 JSON：

- 只記錄事件（筆觸參數、種子值），不記錄像素
- 雙種子確定性系統（strokeSeed + drawingSeed）
- 通常一幅作品的 JSON < 100KB

更進一步，我讓 AI Agent 可以直接生成 JSON 來「畫畫」。AI 不需要操作滑鼠，只要輸出符合格式的 JSON 事件序列，引擎就能還原出完整的水墨畫作。

我用 Claude Code 開發這個專案，搭配 CLAUDE.md 管理專案規範，doc/ 目錄保留歷史版本的 Markdown 技術文檔（TECHNICAL_6~10.md），記錄每次迭代的技術決策。

### 2.3 人機互動：JSON 作為隱性行為記錄

**專案：PolyPaths（植徑集）**

PolyPaths 的概念是：觀眾在介面上畫路徑，以為自己只是在互動，但系統在背後完整記錄了他們的一舉一動。

```
觀眾畫路徑 → 系統記錄 → 壓縮成 hash（放在 URL 裡）→ 解碼還原 JSON → 作為植物生成的骨架
```

一個 URL 裡的 `#data_<hash>` 就能還原完整的繪製路徑。觀眾的行為變成了作品的 DNA。

## 三、我的 LLM 工具鏈：Claude Code + Cursor + Cowork

### 3.1 Claude Code：深度開發與專案管理

Claude Code 是我的主力開發工具。我的使用方式：

- **CLAUDE.md**：每個專案根目錄都放一份，記錄專案規範、技術約束、開發注意事項。Claude Code 每次啟動都會讀取，確保它理解專案脈絡
- **大量使用 .md 規劃**：先用 Markdown 寫大綱、設定、架構，再讓 Claude Code 展開實作
- **技術文檔版本化**：用 doc/ 目錄保留歷史文檔，像 TECHNICAL_6.md 到 TECHNICAL_10.md，記錄每個階段的技術演進

### 3.2 Cursor：即時編碼與快速迭代

Cursor 適合需要快速來回修改的場景，特別是在寫 shader、調整視覺參數、debug 的時候。它的 inline edit 讓我可以邊看程式碼邊跟 LLM 對話。

### 3.3 Cowork 模式：文件處理與日常工作流

Cowork 適合非程式碼的工作——整理文件、生成報告、處理素材。像我現在正在做的這件事：把散落在各處的專案經驗整理成一份教學文件。

## 四、最後一層：用 .md 建立你的數位分身

我做的最後一件事，可能也是最重要的一件事：**把自己結構化。**

我下載了 Facebook、X（Twitter）、Threads 上的所有發言，整理成一份「寫作風格指南」.md，涵蓋：

- **核心人格設定**：身份標籤、人格維度、生涯軌跡
- **寫作語法公式**：不同情境的結構模板（技術體悟型、藝術哲學型、生活廢文型、市場喊話型）
- **關鍵詞彙與口頭禪**：情緒助詞、稱謂、技術術語、標點習慣
- **句型風格**：反問句、極端對比、擬人化

這份文件讓 LLM 可以 index 我的思考方式、表達習慣、價值觀。它不只是助理，更接近一個「模擬我的 agent」——理解我會怎麼想、怎麼說、怎麼決策。

---

## 五、總結：留下錨點，讓一切可以被 index

回顧我從 2021 年到現在的工作方法，核心就是一句話：

> **留下很多錨點，讓它可以被 index，最後透過 meta 生成一個人，以及衍伸的作品。**

| 層級 | 做法 | 目的 |
|------|------|------|
| 作品層 | hash → seed → 確定性生成 | 每件作品可還原、可重播 |
| 記憶層 | JSON 記錄結構（場景/事件/路徑） | 讓 LLM 保持連貫，突破上下文限制 |
| 協作層 | .md 規劃架構，CLAUDE.md 管理規範 | 讓 LLM 理解專案脈絡 |
| 人格層 | 社群發言 → 風格指南 .md | 讓 LLM 理解「我」，成為數位分身 |

結構不是限制，結構是自由。當你把一切都結構化之後，LLM 才能真正成為你的延伸——不只是工具，而是一個理解你、能替你思考的 agent。
