03 — Methodology

核心方法論：JSON 作為記憶層

LLM 的上下文有限、記憶會斷。我的解法：主動建立外部記憶——用 JSON 和 .md 把重要資訊結構化，讓 LLM 每次都能從錨點重新理解脈絡。

用結構化資料（JSON + Markdown）建立人與機器之間的協作協議。
《修仙-七玄關》

04 — Deep Dive · 小說

JSON 實例：故事的外部記憶

寫長篇小說最大的問題是連貫性。LLM 無法記住幾萬字的前文，所以我建了結構化的場景資料庫：

story-structured.json
{
  "chapter_1": {
    "segments": {
      "segment_1": {
        "title": "序曲－雨中的電話",
        "summary": "林承祐在深夜接到命案報告...",
        "content": "（完整文本）",
        "analysis": {
          "setting":        "嘉義市區",
          "characters":     ["林承祐", "小陳"],
          "emotional_tone": "孤獨、麻木、恐懼",
          "foreshadowing":  "宗教案件的陰影重現"
        }
      }
    }
  }
}

每個段落附帶 analysis。LLM 寫第五章時，可回溯所有前面的 analysis，確保情緒曲線連貫、伏筆被正確延續。

關鍵心得：不要期待 LLM 記住一切，主動幫它建索引。
05 — Deep Dive · 水墨

InkField：AI 用 JSON 畫水墨

07 — Toolchain

我的 LLM 工具鏈

Claude Code

深度開發與專案管理

  • CLAUDE.md — 專案規範，每次啟動自動讀取
  • 先用 .md 規劃架構，再展開實作
  • doc/ 目錄保留歷史技術文檔

Cursor

即時編碼與快速迭代

  • 寫 shader、調視覺參數、debug
  • Inline edit 邊看 code 邊跟 LLM 對話
  • 適合快速來回修改的場景

Cowork

文件處理與日常工作流

  • 整理文件、生成報告、處理素材
  • 非程式碼的工作場景
  • 散落經驗 → 結構化教學文件
08 — Digital Twin

最後一層：用 .md 建立數位分身

我把 Facebook、X、Threads 上的所有發言下載下來，整理成一份「寫作風格指南」.md：

核心人格設定
身份標籤、人格維度（焦慮的實踐者、浪漫的工程師、傲嬌的推廣者）、生涯軌跡
寫作語法公式
技術體悟型、藝術哲學型、生活廢文型、市場喊話型——各有結構模板
關鍵詞彙與口頭禪
情緒助詞、稱謂系統、技術術語、標點 / Emoji 習慣
句型風格
反問句、極端對比、擬人化、動漫梗、中英夾雜規則
這份文件讓 LLM 可以 index 我的思考方式、表達習慣、價值觀。
它不只是助理，更接近一個「模擬我的 agent」——理解我會怎麼想、怎麼說、怎麼決策。
09 — Summary

總結：留下錨點，讓一切可以被 index

留下很多錨點，讓它可以被 index，
最後透過 meta 生成一個人，以及衍伸的作品。
層級做法目的
作品層hash → seed → 確定性生成每件作品可還原、可重播
記憶層JSON 記錄結構（場景 / 事件 / 路徑）讓 LLM 保持連貫，突破上下文限制
協作層.md 規劃架構，CLAUDE.md 管理規範讓 LLM 理解專案脈絡
人格層社群發言 → 風格指南 .md讓 LLM 理解「我」，成為數位分身
結構不是限制，結構是自由。

當你把一切都結構化之後，
LLM 才能真正成為你的延伸——
不只是工具，而是一個理解你、能替你思考的 agent。
