01 — Origin

從 2021 年開始的思考：
作品背後都是結構

2021 年，我發表了第一件長篇生成藝術作品 Good Vibrations。表面上是視覺動態藝術——旋轉的圓弧與線段互動，交叉時觸發聲音。

但我在裡面埋了一個隱藏機制：同時按下 A+S+D+F，畫面會切換成「樂譜模式」，揭示每個 token 背後的音樂結構。

tokenData.hash
→
seed
→
確定性隨機數
→
所有參數

每個 token 都是一首獨特的曲子，但只有知道快捷鍵的人才看得到樂譜。

如果作品的本質是結構，那結構本身就可以被記錄、被還原、被重新生成。
這個思考，後來成為我跟 LLM 協作的核心方法論。
02 — Methodology

核心方法論：
JSON 作為記憶層

LLM 的上下文有限、記憶會斷。我的解法不是依賴模型記住一切，而是主動建立外部記憶——用 JSON 和 .md 檔案把所有重要資訊結構化，讓 LLM 每次都能從這些錨點重新理解脈絡。

用結構化資料（JSON + Markdown）建立人與機器之間的協作協議。
《修仙-七玄關》
小說創作
JSON 記錄故事結構——每個段落附帶 analysis（場景、角色、情緒、伏筆），讓 LLM 回溯前文確保連貫性。
JSON 故事資料庫 .md 角色設定 互動式網頁
InkField 墨域
數位水墨引擎
JSON 只記錄事件，不記錄像素。雙種子確定性系統達到 99.9% 重播一致性。AI Agent 可直接生成 JSON 來畫畫。
JSON 事件序列 CLAUDE.md
PolyPaths 植徑集
人機互動生成藝術
觀眾畫路徑，系統在背後完整記錄。路徑壓縮成 hash 放在 URL 裡，解碼後還原為植物生成的骨架。

03 — Deep Dive

JSON 結構實例：
小說的外部記憶

寫長篇小說最大的問題是連貫性。LLM 沒辦法記住幾萬字的前文，所以我建了一套結構：

story-structured.json
{
  "chapter_1": {
    "segments": {
      "segment_1": {
        "title": "序曲－雨中的電話",
        "summary": "林承祐在深夜接到命案報告...",
        "content": "（完整文本）",
        "analysis": {
          "setting": "嘉義市區",
          "characters": ["林承祐", "小陳"],
          "emotional_tone": "孤獨、麻木、恐懼",
          "foreshadowing": "宗教案件的陰影重現"
        }
      }
    }
  }
}

每個段落除了完整文本，還附帶 analysis。當 LLM 寫第五章時，它可以回溯前面所有 analysis，確保情緒曲線連貫、伏筆被正確延續。

關鍵心得：不要期待 LLM 記住一切，主動幫它建索引。
04 — Toolchain

我的 LLM 工具鏈

Claude Code

深度開發與專案管理

  • CLAUDE.md 記錄專案規範，每次啟動自動讀取
  • 先用 .md 規劃架構，再展開實作
  • doc/ 目錄保留歷史技術文檔，記錄迭代演進

Cursor

即時編碼與快速迭代

  • 寫 shader、調整視覺參數、debug
  • Inline edit 邊看程式碼邊跟 LLM 對話
  • 適合需要快速來回修改的場景

Cowork 模式

文件處理與日常工作流

  • 整理文件、生成報告、處理素材
  • 非程式碼的工作場景
  • 把散落的經驗整理成教學文件
05 — Digital Twin

最後一層：
用 .md 建立你的數位分身

我做的最後一件事，可能也是最重要的一件事：把自己結構化。

我下載了 Facebook、X（Twitter）、Threads 上的所有發言，整理成一份「寫作風格指南」.md：

核心人格設定
身份標籤、人格維度（焦慮的實踐者、浪漫的工程師、傲嬌的推廣者）、生涯軌跡
寫作語法公式
技術體悟型、藝術哲學型、生活廢文型、市場喊話型——不同情境的結構模板
關鍵詞彙與口頭禪
情緒助詞、稱謂系統、技術術語、標點與 Emoji 習慣
句型風格
反問句、極端對比、擬人化、動漫梗、中英夾雜規則
這份文件讓 LLM 可以 index 我的思考方式、表達習慣、價值觀。
它不只是助理，更接近一個「模擬我的 agent」。
06 — Summary

總結：留下錨點，讓一切可以被 index

留下很多錨點，讓它可以被 index，
最後透過 meta 生成一個人，以及衍伸的作品。
層級做法目的
作品層hash → seed → 確定性生成每件作品可還原、可重播
記憶層JSON 記錄結構（場景/事件/路徑）讓 LLM 保持連貫，突破上下文限制
協作層.md 規劃架構，CLAUDE.md 管理規範讓 LLM 理解專案脈絡
人格層社群發言 → 風格指南 .md讓 LLM 理解「我」，成為數位分身
結構不是限制，結構是自由。

當你把一切都結構化之後，
LLM 才能真正成為你的延伸——
不只是工具，而是一個理解你、能替你思考的 agent。
